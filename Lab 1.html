Sección Principal

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la red, mientras que WWW es el sistema. WWW usa estándares mientras que Internet es la infraestructura.

¿Cuáles son las partes de un URL?

Protocolo, subdominio, nombre de dominio, dominio de nivel superior, número de puerto, ruta, separador de consulta, parámetros de consulta y fragmento. No necesariamente ocupa todos, pero tiene que tener mínimo 3 partes.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET se utiliza para conseguir datos de un servidor, POST es para crear nuevos recursos, PUT es para reemplazar recursos, PATCH es para actualizarlos, DELETE es para removerlos.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se utiliza el método POST, pues de esta manera los datos se mantienen secretos.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET, pues de esa manera se consigue la dirección de la página a través del código.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que se llevó a cabo la solicitud con éxito. No ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende, pues un error 404 ocurre cuando no se puede encontrar un recurso. El error puede ser causa del usuario al escribir mal la página, pero también puede ser error del desarrollador al eliminar alguna página, configurar mal los enlaces, etc.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, pues el error es que hubo un error interno en el servidor. El usuario no está a cargo del servidor, así que es trabajo del desarrollador

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que se considera obsoleto, y es mala práctica el usarlos en páginas web nuevas. Generalmente es porque existen alternativas mejores. Algunos elementos de HTML4 desaprobados son MENU (ahora se usa UL), BASEFONT, STRIKE, U (estos tres fueron reemplazados por los style sheets)

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 tiene nuevos elementos semánticos (tags), soporte multimedia, y se pueden usar vectores escalables.

¿Qué componentes de estructura y estilo tiene una tabla?

tr, caption, th, thead, td, tbody, colgroup, tfoot, col y border, color, padding

¿Cuáles son los principales controles de una forma HTML5?

Email, color, datetime, teléfono

¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

Generalmente el ciclo de vida es: Identificación de requisitos de software, análisis de requerimientos, diseño, desarrollo, pruebas, implementación y mantenimiento. El ciclo de desarrollo puede ser un modelo en cascada, modelo en espiral o desarrollo ágil (como scrum)
